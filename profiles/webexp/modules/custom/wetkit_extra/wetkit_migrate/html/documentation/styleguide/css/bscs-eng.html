Font size CSS

Introducing CSS classes:

  • font-xsmall
  • font-small
  • font-medium
  • font-large
  • font-xlarge

The CSS provides several font size options that you can use to override the default size. The sizes range from 70% of the default, to 120%, which is the size of an h2.

The different font size styling options
Class Details Example
font-xsmall Extra small font (70%) Resulting text size
font-small Small font (85%) Resulting text size
font-medium Medium (100%) = default size Resulting text size
font-large Large (110%) = same size as an h3 Resulting text size
font-xlarge Extra large (120%) = same size as an h2 Resulting text size

Points of compliance

The following points always apply:

  • Apply these styles as needed to your local content, but don't overuse them.
  • Use font sizes consistently throughout your Web site.

Vertical alignment CSS

Introducing CSS classes:

  • align-top
  • align-middle
  • align-bottom

This CSS adjusts the vertical alignment (top, middle or bottom) of an inline element (an element that does not force extra spacing), and makes it relative to the parent element.

Surprisingly, it is very difficult to vertically align content with pure CSS and this class has limited applications. Typically, this class only works with table cell data (<td>) and images (<img />).

The different vertical alignment styling options
Class Example
align-top
 Resulting text, top vertical alignment
align-middle
 Resulting text, middle vertical alignment
align-bottom
 Resulting text, bottom vertical alignment

Points of compliance

The following point always applies:

  • Apply as needed to your local content, but do not overuse.

Positioning CSS

Items can be layered and stacked on each, however the outer container must be a grid cell. This is due to a CSS technical requirement. The outer item (grid cell) has the attribute of position:relative and the inner item (can be anything) has the attribute of position:absolute. Refer to: Positioning grids

Width CSS

Introducing CSS classes:

  • width-100
  • width-90
  • width-80
  • width-70
  • width60
  • width-50
  • width-40
  • width-30
  • width-20
  • width-10

Since the grid system has its own set of width standards, the below width CSS has little use, other than in tables (<table>) or table headers (<th>). There are rare situations where these classes have a purpose elsewhere.

The different table width styling options
Class Details and examples

Points of compliance

The following points always apply:

  • Apply as needed to your local content, but do not overuse.
  • Use these classes strategically and with caution. White space is an important design feature. However, too much white space makes it difficult to associate related table data when it is overly spread out. The content becomes difficult for the eye to scan, and therefore more difficult to read.
  • Use these classes mainly to control the amount of white space. The browser automatically creates a table that optimizes the content. For the most part it ensures there isn't too much white space between rows or columns, but it isn't foolproof.
