Grid overview

Grids have been around for a long time in print publishing, and have now moved to the Web. A grid is an invisible CSS wire frame that you can use to achieve visual cohesion. It is like a lattice that divides the space into sections where you can place content such as headings, text, lists, images and so on.

Do you have to use grids?

Nope. Grids are not needed when standard content like paragraphs, tables, lists etc is allowed to freely fill the entire content area. Grids are only for creating boxes that need to be positioned and possibly decorated. Some examples include:

  • a right menu
  • a table of contents
  • a grouping of thumbnails
  • anything in columns (except table columns that are generate from the table element)
  • call out boxes
  • alert boxes
  • etc.

For example, the table of contents at the top of this page is in a grid cell, but this entire sub section of information is not. That said, grids to not always have to have a border or any type of decoration at all. So although the table of contents is a visual box, that does not mean all grid cells are so easy to spot. Grid cells are actually invisible by default and decoration can be applied, if desired.

No left menu = 2 choices: 8 or 12 columns

With left menu = Default: 8 columns

Why 8 columns? The addition of a left menu means the remaining columns must divide up to maximize the layouts. It would be a natural assumption to think that a 12 column would provide the most layout outs, but as demostrated below, the remaining columns cannot do a fundamental layout of dividing the space in half. Since this is a basic requirement and can be achived with the 8 columns, then that is the default layout choice.

Both can divide the content area into thirds.

Both can create a right menu the same width as the left menu.

Only one can divide the content area in half.

Optional 12 column container: For complex or alternate layouts, you can call in a grid 12 within the content area. Although the default grid system will likely meet the majority of your needs, this alternate grid version is available and can be easily activated.

A grid 12 living inside of the grid 8.

Grid structure CSS

Introducing CSS classes:

  • span-1
  • span-2
  • span-3
  • span-4
  • span-5
  • span-6
  • clear

When there is a left menu, the default grid system offers a six column layout. This means that each row of grid cells must collectively add up to a total of six. For example, you can have a span-1 + span-5 or you could combine a span-2 + span-2 + span-2.

Grid rows CSS

There are six grid cells (columns) that you can combine within a row of content. To maintain the integrity of each row, you must place a <div class="clear"></div> at the end of each row. This is because grid cells have a float:left property. The clear prevents the next row of grid cells from trying to float up and fill any gaps in the previous row.

Source code: grid row with <div class="clear"></div>

<div class="span-5">
	<p>Text here</p>
</div>
<div class="span-1">
	<p>Text here</p>
</div>
<div class="clear"></div>

Recap: Grid structure and layout options

  • Grids are used to slice up the sections within the content area. They create boxes and/or columns. They are not needed for basic content that is intended to fill the full width of the content area.
  • When there is a left menu, the grid system offers a default of six grid column — each row must total six grids.
  • An optional 12 column grid system can be inserted into the content area
  • Grid cells float to the left, and you can combine them together in a variety of ways to form a row.
  • There are 32 base layout options in the default grid system. There are hundreds more when a grid 12 is added.
  • A single grid cell has a margin-left and margin-right of 10px. This is known as the gutter.
  • A single grid cell has a margin-top of 0px and a margin-bottom of 20px. This can be increased or decreased by adding the margin proxmitity CSS.
  • You need to add <div class="clear"></div> at the end of each row to ensure new rows appear properly.

Rather than generating a border with the border attribute, the CSS calls in a background image of a border. Since a grid cell is a fixed width, the border attribute causes problems. This attribute increases the size of the grid cell width by 2px (1px each for the left and right borders). A background image does not affect the grid width, and it ensures the grid system dimensions are maintained.

Each grid cell size has a different background image. Therefore, you must match the gridX style with the borderX style. For example, use span-1 with border-span-1, span-2 with border-span-2, and so on.

Image CSS and grids

Introducing CSS class:

image-actual

In order to accommodate the two resolutions (800x600 and 1024x768), grids automatically resize themselves to suit the resolution. Text automatically word-wraps — but images are typically a fixed size. This can cause problems within a resizing grid system. Therefore, the CSS controls all images within a grid cell so they expand to 100% of a particular grid's width. However, you can override this feature with the CSS class image-actual.

Elastic images

All three images below use the same single source file which is 224 pixels wide, just like the span-2 cell. The smallest span-1 version pinches the image at the corners; the largest span-3 version stretches, and blurs it. Therefore, the best grid to view the below sunflower image is within in a span-2 at 1024x768 resolution, as it is a perfect fit (optimal size for its placement).

Even though you can use any image to fit any grid cell, it's best to size your image to fit the intended cell. If you recall from the above topic Grid building blocks, a span-2 cell for example, ranges from 194 pixels to 224 pixels wide, depending on the user's resolution. To reduce visual distortion, ensure you size the image to match the grid cell's larger 1024x768 width, not the 800x600 width.

Fixed width images

If you do not want an image to stretch to 100% of the cell width, you can apply the image-actual class to the image. For example, if you view the 20x20 pixel check-mark image at 100% of a grid cell, it appears pixilated. The image-actual class ensures the image retains its width and height properties, and never visually distorts.

Source code: Fluid image (100% of grid cell width)

<div class="span-1">
	<img src="tick.gif" alt="___" width="20" height="20" />
	<img src="x.gif" alt="___" width="20" height="20" />
</div>

Source code: Fixed image size

<div class="span-1">
	<img class="image-actual" src="tick.gif" alt="___" width="20" height="20" />
	<img class="image-actual" src="x.gif" alt="___" width="20" height="20" />
</div>

Recap: Images and grids

  • Images within a grid automatically stretch to fill the width of that grid cell.
  • If you do not want an image to stretch to 100% of the cell width, apply the image-actual class to the image.
  • When you optimize your image, size it to fit the grid cell width you are most likely going to place it in. Size the image to match the larger grid cell 1024x768 width. Refer to Grid building blocks for sizing details.

Proximity CSS (margins, padding, gutters) and grids

Introducing CSS classes:

  • margin-bottom-none
  • margin-bottom-small
  • margin-bottom-medium
  • margin-bottom-large
  • margin-bottom-xlarge
  • margin-top-none
  • margin-top-small
  • margin-top-medium
  • margin-top-large
  • margin-top-xlarge

White space is an important feature of the grid system — which automatically generates by default. You can overwrite some white space with additional CSS in order to increase or decrease the default settings.

Within a grid system, you can only change the white space above and below a grid cell. You can never change the gutters (white space to the left and right of a grid cell). This is because the grid system design fits perfectly in the content area, and is tied to the user's browser setting. There is no room for extra pixels. Reducing pixels causes the cells to behave in unexpected ways due to their float: left property.

Padding

Paragraphs, lists, and headings come with padding; grid cells do not have inside padding. The most common mistake is to place text inside a grid cell, but forget to wrap it in something like a <p> tag.

With a paragraph tag — text is easier to read with the padding.

<div class="span-2 background-light">
	<p>With paragraph tag</p>
</div>
No paragraph tag (not ideal) 
<div class="span-2 background-light">
	Forgot paragraph tag (not ideal)
</div>

Margins and gutters

The white space to the left and right of a grid cell is called a gutter (although it is just a margin-right in the CSS). The default settings of a grid cell are:

  • Default margin-top is set to zero.
  • Default margin-right, which creates the gutter, has two settings. A user with an 800x600 screen resolution has a gutter of 10px. A user with a screen resolution of 1024x768 or higher has a gutter of 20px.
  • Default margin-bottom is set to 20 pixels.
  • Default margin-left is set to zero.

You can only increase or decrease vertical proximity of content (move up or down). There are a variety of reasons to modify vertical proximity, such as visually connecting related components. In which case, you may need to increase or decrease white space between menu items, or move content closer to or farther from a heading tag. The classes you can use are as follows:

Bottom proximity CSS

  • 0px margin-bottom = class margin-bottom-none
  • 2px margin-bottom = class margin-bottom-small
  • 10px margin-bottom = class margin-bottom-medium
  • 20px margin-bottom = class margin-bottom-large
  • 50px margin-bottom = class margin-bottom-xlarge

Top proximity CSS

  • 0px margin-bottom = class margin-top-none
  • 2px margin-top = class margin-top-small
  • 10px margin-top = class margin-top-medium
  • 20px margin-top = class margin-top-large
  • 50px margin-top = class margin-top-xlarge

If you need to modify the vertical proximity of grid cells, you can apply the margin-bottom-(x) class to the upper cell. This overrides the margin-bottom from the default of 20px with your new value, and moves the content in the lower cell up or down (depending on the class you apply).

You can also apply the margin-top-(x) class to the lower cell. Each situation dictates whether you need to apply the margin-bottom-(x) or margin-top-(x) class to the content. Refer to the examples below:

Recap: Proximity: Margins, padding and gutters

  • White space is an important feature of the grid system.
  • A gutter is the white space to the left and right of a grid cell.
  • You can only change the white space above and below a grid cell — not the gutters.
  • Grid cells do not have inside padding. The padding is inherent to a paragraph, list or heading that you place inside a grid cell.

Grid parents and children CSS

Introducing CSS classes:

  • row-start
  • row-end

Within a grid cell, you can place other grids, thereby creating a grid parent that contains grid children (like a table inside a table). Grids are typically rows of content, read from left to right. However, there are times when you need to cluster and group content in a customized way. Grid parents allow you to do this.

Placing grids inside grids allows you to stack content and create unique layouts, while adhering to the structure and rigour of a grid system. Additionally, JAWS (software for visually impaired staff) can read grouped content within the grid parent before moving to the next grid cell, even when you vertically stack this content. This ensures both visual and visually-impaired users have the same experience.

row-start and row-end CSS

In each row, you need to remove the left hand gutter from the first child and the right hand gutter from the last grid child so it fits into the available space (the margin-right is either 5px or 10px, depending on the user's resolution). Apply the row-start to the first child in the row and then row-end CSS to the last child in a given row to achieve this. Refer to the example below.

Recap: Grid parents and grid children

  • Within a grid cell, you can place other grids, thereby creating a grid parent that contains grid children (like a table inside a table).
  • In each row, you need to remove the left hand gutter from the first child and the right hand gutter from the last grid child so it fits into the available space (the margin-right is either 5px or 10px, depending on the user's resolution). Apply the row-start to the first child in the row and then row-end CSS to the last child in a given row to achieve this.
  • You have unlimited layout options when you use a parent / child grid system.

Equal height columns

Introducing CSS class:

equalize

Historically, you used tables for layout, and it was easy to create columns of equal height. In every row, the table data cells always assumed the height of the tallest cell in that row. CSS grid cells create similar rows of content, but because they are not associated with each other, they do not apply borders and background colour fills in the same way. Without JavaScript, there is no way for the browser to know what the tallest grid cell per row cell is. Therefore, a CSS / JavaScript solution is necessary to achieve this same effect.

By using equalize on the grid parent, its row of grid children automatically assume the height of the tallest child.

Float versus align

Float CSS makes the grid cell float, but does not affect the content inside inside it. Align CSS affects only the content within the grid cell, but not the grid cell itself. For example:

  • <div class="span-2 float-right">content</div> moves the grid cell to the right side of the page, and
  • <div class="span-2 align-right">content</div> moves the content within the cell to the right side of the cell, but the cell itself does not move.

float-right with a grid cell

Grid cells behave like fixed width columns, so content never freely floats in the available white space below or around a neighbouring cell. However, there are lots of situations where you want the dimensions and structure of a grid, but need the benefit of regular free-flowing content. For example, if you want a small floating box on the right side of your page, and you want the body text to fill the space around it. You need to do the following:

  1. Create the grid cell you want to appear on the right. A span-2 is ideal for this.
  2. Apply the float-right class to this grid cell.
  3. Do not put the rest of the content in a grid cell, like a span-4. Instead, treat it like regular body text. The result is non-restricted content that fills the available space.

Recap: Indent, float and align

  • You can use empty grid cells to create white space such as indents.
  • You can use float and alignment CSS to further customize the grid system.
  • Float CSS floats the grid cell, but doesn't affect the content inside the grid. Align CSS affects only the content inside the grid cell — not the grid cell itself. For example:
    • <div class="span-2 float-right">content</div> moves the grid cell to the right side, and
    • <div class="span-2 align-right">content</div> moves the content within the cell to the right side of the cell. The grid cell doesn't move.

Print CSS and grids

Introducing CSS classes:

  • print-span-1
  • print-span-2
  • print-span-3
  • print-span-4
  • print-span-5
  • print-span-6
  • print-span-7
  • print-span-8

Grid cells do not keep their design properties when printed, for various reasons:

  • Browsers do not process complex CSS well when printing, so grids print differently in IE6 versus IE8, both with unexpected and undesirable results. 
  • To help minimize paper usage, print versions use minimal formatting by maximizing the full width and height of a page

However, if it is crucial to maintain a grid layout, several print classes have been created:

  • print-span-1
  • print-span-2
  • print-span-3
  • print-span-4
  • print-span-5
  • print-span-6
  • print-span-7
  • print-span-8
<div class="span-5 border-span-5 print-span-5">

Adding these classes have no affect on the Web page itself, but will keep the grid layout when the page is printed.  There is rarely a need for this, but may be handy for directives or policy documents.

Note: These classes will not work on grid parent/child relationships.  They are for simple grid designs.
